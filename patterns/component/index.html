Badge

The badge is used for any image that represents a brand, identity, accolade, status or similar marking.

Badge - standard

Show code: 
<a href="https://www.npeu.ox.ac.uk/ctu" class="c-badge  c-badge--limit-height">
    <img src="/assets/images/brand-logos/unit/npeu-ctu-logo.svg" onerror="this.src='/assets/images/brand-logos/unit/npeu-ctu-logo.png'; this.onerror=null;" alt="Logo: NPEU CTU" height="80">
</a>

Every effort should be made to provide a badge as an SVG. The build process will generate a fallback PNG. So that this PNG is of a reasonable quality, SVG's should be around 300px high and include width and height attributes initially. SVGO removes these attributes for the final SVG output, but svg2png requires that they're present when it runs.

By default, badges are intrinsically sized, meaning they'll be a small as possible. Without a height attribute on the img tag, or a CSS (min-)height, the badges will collapse to nothing, so make sure one of those is in place.

Badges that are part of lists or other layout should probably try to fill the container (possibly with some max-height cap). WIP: it's probably up to the layout to force it's contents to fill it's space but this is tricky. Separation of concerns etc. May opt for a c-badge-fill-container class?

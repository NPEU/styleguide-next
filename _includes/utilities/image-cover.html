Image Cover

Image covers should make use of the min-height classes to ensure imamges don't collapse to nothing in certain contexts. The default min-heights are in steps of 10% sprinkled with common other picture ratios.

Note the max-width set on the containers below are inline styles and are not part of the patter. They're there to help make the images more manageable in this context.

Also note it's important to include a sensible width attribute, and a height attribute that corresponds to the images aspect ratio. This helps keep the images manageable in a FallBack scenario, and helps browsers reserve space for images to avoid layout shift.

Classic 16:9 min height (56.25%)

Show code: 
<div class="u-image-cover  js-image-cover  u-image-cover--min-56-25">
    <div class="u-image-cover__inner">
        <img src="/assets/images/patterns/placeholder-1--small.jpg" sizes="100vw" srcset="/assets/images/patterns/placeholder-1--large.jpg 1600w, /assets/images/patterns/placeholder-1--medium.jpg 900w, /assets/images/patterns/placeholder-1--small.jpg 300w" alt="" class="u-image-cover__image" width="200" height="133">
    </div>
</div>
